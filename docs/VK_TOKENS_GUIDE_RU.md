# OpenClaw VK Plugin — Официальная документация SDK

## 1. Обзор

OpenClaw VK Plugin обеспечивает программное взаимодействие с VK API от имени сообщества.

Возможности:

- Получение входящих событий (Bots Long Poll / Callback API)
- Отправка сообщений (текст, клавиатуры)
- Загрузка медиафайлов (изображения, документы)
- Публикации на стене
- Работа с VK Market

---

## 2. Требования

### Обязательные параметры

| Параметр | Тип | Описание |
|----------|-----|----------|
| `token` | `string` | Токен доступа сообщества (Community Access Token) |
| `groupId` | `string` | Числовой идентификатор сообщества |

---

## 3. Аутентификация

### 3.1 Тип токена

Плагин использует **Community Access Token** — токен доступа сообщества.

Формат токена:
```
vk1.a.<token>
```

### 3.2 Права токена сообщества

Токен **ДОЛЖЕН** иметь минимальный набор прав:

| Scope | Обязательный | Назначение |
|-------|-------------|-----------|
| `messages` | ✅ Да | Получение и отправка сообщений |

Дополнительные права (согласно официальной документации VK API):

| Scope | Назначение |
|-------|-----------|
| `photos` | Загрузка изображений |
| `docs` | Загрузка файлов |
| `stories` | Публикация stories |
| `app_widget` | Виджеты приложений сообщества |
| `manage` | Расширенные методы API (статистика, Callback) |

> **Важно о `wall` и `market`:** Эти scope **отсутствуют** в токене сообщества. Публикации на стену доступны через `manage`. Методы VK Market (`market.get`, `market.edit` и др.) требуют отдельного **пользовательского токена** (User Access Token) — см. раздел 3.3.

### 3.3 Пользовательский токен (User Access Token) для VK Market

Все методы `market.*` (market.get, market.edit, market.add и др.) работают **только с пользовательским токеном** — токен сообщества для них не подходит.

Для получения пользовательского токена с доступом к Market:

1. Создайте приложение на [vk.com/apps?act=manage](https://vk.com/apps?act=manage)
2. Запросите права `market` (scope: `134217728`) через OAuth Implicit Flow:
   ```
   https://oauth.vk.com/authorize?client_id=ВАШ_APP_ID&display=page
     &redirect_uri=https://oauth.vk.com/blank.html
     &scope=134217728&response_type=token&v=5.199
   ```
3. Токен содержится в URL после редиректа: `#access_token=vk1.a...`

> **Важно:** Scope `market` для новых приложений, созданных в Сервисе авторизации VK ID, требует явного одобрения. Напишите запрос на `devsupport@corp.vk.com`, указав ID приложения и сообщества. Подробнее: [dev.vk.com/reference/access-rights](https://dev.vk.com/reference/access-rights).

---

### 3.4 Жизненный цикл токена

Токен:

- Не имеет фиксированного срока действия
- Остаётся активным до тех пор, пока:
  - Не будет отозван вручную
  - Не изменятся права доступа
  - Не произойдёт сброс настроек безопасности

> **Важно:** Токен отображается только один раз после создания. Повторный просмотр невозможен. При утере необходимо создать новый токен.

### 3.4 Требования безопасности

### 3.5 Требования безопасности

**ДОЛЖЕН:**
- Храниться в защищённом хранилище (переменные окружения или secret manager)
- Доступ к конфигурационным файлам должен быть ограничен
- Токен не должен попадать в логи или клиентский код

**НЕ ДОЛЖЕН:**
- Коммититься в систему контроля версий
- Передаваться в открытом виде
- Встраиваться во фронтенд-приложения

## 4. Настройка сообщества

Перед использованием плагина выполните настройку сообщества в интерфейсе VK.

### 4.1 Включить Bots Long Poll API

```
Управление → Сообщения → Настройки для ботов → Включить Long Poll API
```

### 4.2 Разрешить отправку сообщений

```
Управление → Сообщения → Настройки для ботов → Разрешить сообщения от имени сообщества
```

### 4.3 Рекомендации для продакшена

- Long Poll API подходит для разработки и небольших нагрузок
- Для продакшен-окружений **РЕКОМЕНДУЕТСЯ** использовать **Callback API** (вебхуки):
  ```
  Управление → Сообщения → Callback API → Укажите URL сервера
  ```

## 5. Group ID (ID сообщества)

### 5.1 Определение

`groupId` — числовой идентификатор сообщества ВКонтакте.

### 5.2 Ограничения

- **ДОЛЖЕН** быть числом
- **НЕ ДОЛЖЕН** содержать знак минус
- **НЕ ДОЛЖЕН** быть коротким именем (screen name)

### 5.3 Получение

**Способ 1 — через интерфейс VK:**
```
Управление → Работа с API → поле «ID сообщества»
```
Если поле отсутствует: **Управление → Общее → Адрес страницы** — используйте число после `club`.

**Способ 2 — через API:**

```bash
curl "https://api.vk.com/method/groups.getById?group_id=SHORT_NAME&access_token=TOKEN&v=5.199"
```

Параметры запроса:

| Параметр | Обязательный |
|----------|-------------|
| `group_id` | ✅ Да |
| `access_token` | Опционально |
| `v` | ✅ Да |

Пример ответа:
```json
{
  "response": [
    {
      "id": 123456789,
      "name": "Название сообщества"
    }
  ]
}
```

Используйте значение `id`.

## 6. Конфигурация

### 6.1 JSON-конфигурация

Файл: `openclaw.json`

```json
{
  "plugins": {
    "entries": {
      "vk": {
        "token": "vk1.a.YOUR_TOKEN",
        "groupId": "123456789"
      }
    }
  }
}
```

> `groupId` передаётся как строка — это ожидаемый формат плагина.

### 6.2 Переменные окружения (рекомендуется)

```env
VK_TOKEN=vk1.a.YOUR_TOKEN
VK_GROUP_ID=123456789
```

### 6.3 Настройка через интерфейс

```
Settings → Channels → VK → Configure
```

## 7. Поведение в рантайме

### 7.1 Получение событий

Плагин поддерживает два режима:

| Режим | Описание |
|-------|----------|
| Long Poll | Pull-модель получения событий |
| Callback API | Webhook-доставка событий |

### 7.2 Поток обработки сообщений

```
Событие получено от VK
        ↓
Событие разобрано
        ↓
Обработчик выполнен
        ↓
Ответ отправлен через API
```

---

## 8. Проверка работоспособности

### 8.1 Функциональная проверка

Отправьте сообщение в сообщество.

Ожидаемое поведение:
1. Плагин получает событие
2. Формируется ответ
3. Ответ отправляется пользователю

### 8.2 Проверка через API

```bash
curl "https://api.vk.com/method/groups.getById?group_id=123456789&access_token=vk1.a.TOKEN&v=5.199"
```

Ожидаемый ответ:
```json
{
  "response": [
    {
      "id": 123456789,
      "name": "Название сообщества"
    }
  ]
}
```

## 9. Обработка ошибок

### 9.1 Коды ошибок

| Код | Описание | Действие |
|-----|----------|----------|
| `5` | Неверный токен | Создать новый токен |
| `7` | Недостаточно прав | Проверить scope токена |
| `10` | Внутренняя ошибка VK | Повторить с задержкой |
| `15` | Доступ запрещён | Проверить права токена |
| `18` | Пользователь или сообщество удалён/заблокирован, либо токен недействителен | Проверить статус сообщества, создать новый токен |
| `100` | Неверный параметр | Проверить `groupId` и параметры запроса |

### 9.2 Стратегия повторных запросов

Рекомендуется:
- Экспоненциальный backoff
- Максимум попыток: 3–5
- С джиттером

---

## 10. Лимиты запросов

VK API устанавливает лимиты на количество запросов на токен.

Рекомендации:
- Реализовать троттлинг запросов
- Ставить исходящие запросы в очередь
- Избегать пиковой нагрузки

---

## 11. Рекомендации для продакшена

**ДОЛЖНО:**
- Использовать Callback API для продакшен-нагрузок
- Реализовать логику повторных запросов
- Логировать все API-взаимодействия (без чувствительных данных)
- Мониторить частоту ошибок

**РЕКОМЕНДУЕТСЯ:**
- Использовать отдельные токены для каждого окружения (dev / staging / production)
- Регулярно ротировать токены
- Настроить алертинг на всплески ошибок

---

## 12. Безопасность

### 12.1 Хранение токена

Использовать:
- Переменные окружения
- Secret manager (Vault, AWS Secrets Manager и др.)

### 12.2 Управление доступом

- Ограничить права доступа к конфигурационным файлам
- Изолировать рантайм-окружение
- Не использовать общие учётные данные между сервисами

### 12.3 Реагирование на инцидент

При компрометации токена:

1. Немедленно отозвать токен
2. Выпустить новый токен
3. Обновить все конфигурации
4. Проверить логи на предмет несанкционированного использования

---

## 13. Ограничения

- Long Poll может терять события при нестабильном сетевом соединении
- Callback API требует публично доступного HTTPS-эндпоинта
- Поведение VK API может изменяться между версиями — проверяйте после обновлений
- Методы VK Market недоступны через токен сообщества — требуется пользовательский токен (User Access Token) с scope `market`
- Scope `market` для новых VK ID приложений выдаётся только после одобрения поддержкой VK (devsupport@corp.vk.com)

---

## 14. Версионирование API

Во всех примерах используется:
```
v=5.199
```

Рекомендуется использовать последнюю стабильную версию VK API. Актуальный список: [dev.vk.com/versions](https://dev.vk.com/versions).
